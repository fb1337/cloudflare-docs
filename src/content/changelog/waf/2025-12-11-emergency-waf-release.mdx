The previously deployed rules targeting server-function exposure and resource-exhaustion patterns have been updated for accuracy and CVE alignment. Their descriptions now read:

- “React – Leaking Server Functions – CVE:CVE-2025-55183”
- “React – DoS – CVE:CVE-2025-55184”

These changes formally map the existing detections to the newly disclosed React Server Components vulnerabilities. No modifications were made to the underlying detection logic or signal quality.

**Key Findings**

React server-function exposure and resource-exhaustion rules have been updated to reflect CVE-2025-55183 and CVE-2025-55184. 
The ruleset now explicitly references CVE-2025-55183 and CVE-2025-55184

**Impact**

These updates enhance protection against exploitation attempts targeting React Server Components by reinforcing detection around server-function leakage and resource-exhaustion techniques. They also improve visibility into misuse patterns that may expose internal logic or degrade service availability. As these vulnerabilities are newly disclosed, the updated rule descriptions ensure the protections are now clearly aligned with the latest CVE information and current threat landscape.
